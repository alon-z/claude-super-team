  <name>Task 1: Create json-sync.sh with MD-to-JSON conversion functions</name>
  <files>plugins/claude-super-team/scripts/json-sync.sh</files>
  <action>
Create `plugins/claude-super-team/scripts/json-sync.sh` as an executable bash script. This script converts existing .planning/ MD files into their JSON counterparts and validates the output.

**Script structure:**

1. **Header and setup:**
   - Shebang: `#!/usr/bin/env bash`
   - Description comment: "json-sync.sh - Convert .planning/ MD files to JSON and validate structure"
   - Set `P="${P:-.planning}"` (same pattern as gather-common.sh)
   - Check jq availability: `command -v jq >/dev/null 2>&1 || { echo "ERROR: jq is required but not installed"; exit 1; }`

2. **Atomic write helper function `write_json()`:**
   ```bash
   write_json() {
     local target="$1"
     local content="$2"
     local tmp="${target}.tmp"
     if echo "$content" | jq -e '.' > "$tmp" 2>/dev/null; then
       mv "$tmp" "$target"
       return 0
     else
       rm -f "$tmp"
       echo "ERROR: Invalid JSON for $target"
       return 1
     fi
   }
   ```

3. **`sync_project()` function** -- converts PROJECT.md to PROJECT.json:
   - Extract projectName from the first `# ` heading
   - Extract description from the paragraph below "## What This Is" (or the first non-heading paragraph)
   - Extract coreValue from "**Core value:**" or "Core value:" line
   - Extract requirements.validated from `### Validated` section (lines matching `- ✓` or `- [x]`)
   - Extract requirements.active from `### Active` section (lines matching `- [ ]`)
   - Extract requirements.outOfScope from `### Out of Scope` section (lines matching `- `)
   - Extract constraints from `## Constraints` section (lines matching `- `)
   - Extract decisions from the Key Decisions table (pipe-delimited rows, skip header/separator)
   - Extract preferences.executionModel from `## Preferences` section (grep `execution-model:`)
   - Use awk to extract each section, pipe through `jq --arg` / `jq -Rs` for safe string construction
   - Build the complete JSON object using `jq -n` with `--arg` for scalars and `--argjson` for arrays/objects
   - Schema:
     ```json
     {
       "projectName": "string",
       "description": "string",
       "coreValue": "string",
       "requirements": {
         "validated": [{"text": "string"}],
         "active": [{"text": "string"}],
         "outOfScope": ["string"]
       },
       "constraints": ["string"],
       "decisions": [{"decision": "string", "rationale": "string", "outcome": "string"}],
       "preferences": { "executionModel": "string" },
       "lastUpdated": "string"
     }
     ```
   - Call `write_json "$P/PROJECT.json" "$json"`

4. **`sync_roadmap()` function** -- converts ROADMAP.md to ROADMAP.json:
   - Extract overview from `## Overview` section (first paragraph)
   - Extract phases array from `## Phases` section checkbox list. For each `- [x]` or `- [ ]` line:
     - Parse id (Phase number), complete (checkbox state), name (text after "Phase N: ")
   - Extract phase details from `### Phase N:` blocks. For each block extract:
     - goal (from `**Goal**:` line)
     - dependsOn (from `**Depends on**:` line, parse phase numbers)
     - successCriteria (from `**Success Criteria**` section, numbered list items)
     - sprint (from `**Sprint**:` line)
     - size (from `**Size**:` line)
   - Merge phases list with details into a single phases array
   - Extract sprints from `## Sprint Summary` table
   - Extract progress from `## Progress` table
   - Use the awk-pipe-to-jq pattern from RESEARCH.md for list/table extraction
   - Schema:
     ```json
     {
       "overview": "string",
       "phases": [{"id": "string", "name": "string", "complete": boolean, "goal": "string", "dependsOn": ["string"], "successCriteria": ["string"], "sprint": number|null, "size": "string"|null}],
       "sprints": [{"id": number, "phases": ["string"], "description": "string"}],
       "progress": [{"phase": "string", "status": "string", "completed": "string"}],
       "lastUpdated": "string"
     }
     ```
   - Call `write_json "$P/ROADMAP.json" "$json"`

5. **`sync_state()` function** -- converts STATE.md to STATE.json:
   - Extract currentPosition.phase from `Phase:` line (number)
   - Extract currentPosition.status from `Status:` line
   - Extract currentPosition.lastActivity from `Last activity:` line
   - Extract currentPosition.focus from `**Current focus:**` line
   - Extract preferences: grep for `execution-model:`, `simplifier:`, `verification:` lines. Store with kebab-case keys (not camelCase) to match emit_preferences output format
   - Extract active decisions from `### Decisions` section, stopping at `### Decision Archive` (use existing awk pattern: `awk '/^### Decision Archive/ { exit }'`)
   - Extract blockers from `### Blockers/Concerns` section
   - Schema:
     ```json
     {
       "currentPosition": {"phase": number, "status": "string", "lastActivity": "string", "focus": "string"},
       "preferences": {"execution-model": "string", "simplifier": "string", "verification": "string"},
       "decisions": [{"text": "string"}],
       "blockers": ["string"],
       "lastUpdated": "string"
     }
     ```
   - Call `write_json "$P/STATE.json" "$json"`

6. **`sync_ideas()` function** -- converts IDEAS.md to IDEAS.json:
   - Parse by session boundary (`# Brainstorming Session:` heading)
   - For each session: extract title, date, mode
   - Extract ideas from `### Idea N:` blocks: name, description, decision status
   - Extract approved table from `## Approved Ideas Summary`
   - Extract deferred list from `## Deferred Ideas`
   - Extract rejected list from `## Rejected Ideas`
   - Schema:
     ```json
     {
       "sessions": [{"title": "string", "date": "string", "mode": "string", "ideas": [{"id": number, "name": "string", "decision": "string", "priority": "string", "description": "string"}], "approved": [{"name": "string", "priority": "string", "nextStep": "string"}], "deferred": [{"name": "string", "reason": "string"}], "rejected": ["string"]}],
       "lastUpdated": "string"
     }
     ```
   - Call `write_json "$P/IDEAS.json" "$json"`

7. **`sync_all()` function:**
   - Check `$P` directory exists, exit with error if not
   - Track results: created count, skipped count, error count
   - For each file type (PROJECT, ROADMAP, STATE, IDEAS):
     - If the MD file exists, call the sync function
     - If the MD file does not exist, skip silently
   - Print summary: "Synced: N files | Skipped: N | Errors: N"
   - List each file created/updated with size

8. **`validate_json()` function:**
   - Takes a JSON file path
   - Checks it exists and is valid JSON with `jq -e '.' "$file" >/dev/null 2>&1`
   - Returns 0/1

9. **Main entry point:**
   - If called with `--all` or no args: run `sync_all`
   - If called with a specific file name (project, roadmap, state, ideas): run that sync function
   - If called with `--validate`: validate all existing JSON files
   - Add `--help` flag for usage

Make the script executable: `chmod +x json-sync.sh`

**Critical implementation details:**
- All awk patterns must be resilient to missing sections (output empty arrays/nulls, not errors)
- Use `jq -n --arg` for every string value (never shell interpolation into jq filters)
- Handle edge case where MD file exists but section is empty (produce empty arrays, not null)
- For the lastUpdated field: extract from `_Last updated:` footer line if present, otherwise use empty string
- Preferences must use kebab-case keys in JSON (e.g., `"execution-model"`, NOT `"executionModel"`) to avoid needing key conversion in emit_preferences

**Step 5: Update emit_preferences():**

This is a prime candidate for JSON-first path. Currently it greps for specific keys in STATE.md. Update to:

```bash
emit_preferences() {
  echo "=== PREFERENCES ==="
  if [ "$_JQ_AVAILABLE" = "true" ] && [ -f "$P/STATE.json" ]; then
    # JSON path: extract preferences with kebab-case keys
    jq -r '.preferences // {} | to_entries[] | "\(.key): \(.value)"' "$P/STATE.json" 2>/dev/null && {
      [ "${CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS:-}" = "1" ] && echo "teams-available: true" || echo "teams-available: false"
      return 0
    }
  fi
  # MD fallback (existing logic)
  grep -E '^execution-model:' "$P/STATE.md" 2>/dev/null || echo "execution-model: unset"
  grep -E '^simplifier:' "$P/STATE.md" 2>/dev/null || echo "simplifier: unset"
  grep -E '^verification:' "$P/STATE.md" 2>/dev/null || echo "verification: unset"
  [ "${CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS:-}" = "1" ] && echo "teams-available: true" || echo "teams-available: false"
}
```

Note: The `&& { ...; return 0; }` pattern ensures the MD fallback only executes if jq extraction fails. The JSON must have kebab-case preference keys so `to_entries` produces output like `execution-model: opus` which matches the current grep-based output exactly.

**Step 6: Update emit_sync_check():**

Currently uses grep/awk to extract phase numbers from directories, ROADMAP.md, and STATE.md. Update with JSON-first path:

```bash
emit_sync_check() {
  echo "=== SYNC_CHECK ==="

  # DIR_PHASES: always from filesystem (source of truth)
  echo -n "DIR_PHASES: "
  if [ -d "$P/phases" ]; then
    for dir in "$P"/phases/*/; do
      [ -d "$dir" ] || continue
      basename "$dir" | sed 's/^\([0-9.]*\)-.*/\1/' | sed 's/^0*//'
    done 2>/dev/null | sort -V | tr '\n' ' '
  fi
  echo

  # ROADMAP_PHASES: try JSON first
  echo -n "ROADMAP_PHASES: "
  if [ "$_JQ_AVAILABLE" = "true" ] && [ -f "$P/ROADMAP.json" ]; then
    jq -r '[.phases[].id] | sort_by(tonumber? // .) | .[]' "$P/ROADMAP.json" 2>/dev/null | tr '\n' ' ' && echo && true
  else
    grep -oE 'Phase [0-9]+(\.[0-9]+)?' "$P/ROADMAP.md" 2>/dev/null | awk '{print $2}' | sort -V | uniq | tr '\n' ' '
    echo
  fi

  # STATE_PHASE: try JSON first
  echo -n "STATE_PHASE: "
  if [ "$_JQ_AVAILABLE" = "true" ] && [ -f "$P/STATE.json" ]; then
    jq -r '.currentPosition.phase // empty' "$P/STATE.json" 2>/dev/null
  else
    grep -E '^Phase:' "$P/STATE.md" 2>/dev/null | head -1 | grep -oE '[0-9]+(\.[0-9]+)?' | head -1
  fi
  echo

  # CHECKED/UNCHECKED: try JSON first
  if [ "$_JQ_AVAILABLE" = "true" ] && [ -f "$P/ROADMAP.json" ]; then
    jq -r '.phases[] | select(.complete == true) | "CHECKED: \(.id)"' "$P/ROADMAP.json" 2>/dev/null
    jq -r '.phases[] | select(.complete == false) | "UNCHECKED: \(.id)"' "$P/ROADMAP.json" 2>/dev/null
  else
    grep -E '^\s*- \[x\] Phase' "$P/ROADMAP.md" 2>/dev/null | grep -oE 'Phase [0-9]+(\.[0-9]+)?' | awk '{printf "CHECKED: %s\n", $2}'
    grep -E '^\s*- \[ \] Phase' "$P/ROADMAP.md" 2>/dev/null | grep -oE 'Phase [0-9]+(\.[0-9]+)?' | awk '{printf "UNCHECKED: %s\n", $2}'
  fi
}
```
