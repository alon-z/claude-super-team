**troubleshooting.md updates:**

1. **Add new "JSON Layer" section** after the "Telemetry and Metrics" section:

   ```markdown
   ### JSON Layer

   #### "JSON files out of sync with MD"

   **Symptom:** Gather scripts return stale data even though MD files were updated

   **Cause:** Manual MD edit without regenerating JSON

   **Solution:**
   ```
   /cst-help migrate
   ```
   This regenerates all JSON files from current MD content. Safe to run anytime.

   #### "jq not found"

   **Symptom:** Gather scripts fall back to MD parsing even when JSON files exist

   **Cause:** jq not installed on the system

   **Solution:**
   - Install jq: `brew install jq` (macOS) or `apt install jq` (Ubuntu/Debian)
   - Gather scripts work fine without jq (silent fallback to MD parsing)
   - JSON files are still useful for /cst-help migrate and manual queries

   #### "JSON file is empty or corrupt"

   **Symptom:** `jq: parse error` when reading .planning/*.json

   **Cause:** Interrupted write or manual corruption

   **Solution:**
   - Delete the corrupt JSON file
   - Run `/cst-help migrate` to regenerate
   - All JSON writes use atomic pattern (write to .tmp then mv) to prevent this

   #### "Which files should I commit?"

   **Symptom:** Unsure whether to commit JSON files alongside MD files

   **Solution:**
   - Yes, commit both MD and JSON files
   - They are always in sync when produced by skills
   - Treat JSON files as derived artifacts (can always be regenerated from MD)
   ```
