  <name>Task 1: Create phase-utils.sh with normalize_phase() and find_phase_dir()</name>
  <files>plugins/claude-super-team/scripts/phase-utils.sh</files>
  <action>
Create a bash script at `plugins/claude-super-team/scripts/phase-utils.sh` that is designed to be sourced by skills (not executed directly). It must provide two functions:

**`normalize_phase()`**: Accepts a raw phase number as $1 (e.g., "2", "2.1", "02", "02.1", "10", "10.3"). Returns the zero-padded formatted phase string via stdout. Logic:
- If the input contains a dot: split on `.`, zero-pad the integer part to 2 digits with `printf "%02d"`, leave the decimal part as-is, join with `.`. Example: `2.1` -> `02.1`, `10.3` -> `10.3`.
- If no dot: zero-pad to 2 digits. Example: `2` -> `02`, `10` -> `10`.
- Already-padded input should pass through unchanged: `02` -> `02`, `02.1` -> `02.1`.

**`find_phase_dir()`**: Accepts a normalized phase string as $1 (e.g., "02", "02.1"). Prints the matching directory path under `.planning/phases/` via stdout. Logic:
- Use `ls -d .planning/phases/${1}-* 2>/dev/null | head -1` to find the directory.
- If no match found, print nothing (caller handles the error).

Include a header comment explaining this is a shared utility sourced by skills via `source "${CLAUDE_PLUGIN_ROOT}/scripts/phase-utils.sh"`. Make the script `chmod +x` compatible but note it's designed for sourcing.

Edge case to handle explicitly: input like "2.10" must produce "02.10" (not "02.1" -- the decimal part is preserved verbatim).
  </action>
