  <action>
Create the gather-data.sh script at `plugins/claude-super-team/skills/metrics/gather-data.sh`. Follow the exact pattern from existing gather scripts (progress, drift).

The script must:

1. Source gather-common.sh: `source "$(dirname "$0")/../../scripts/gather-common.sh"`

2. Emit standard sections using shared functions: `emit_project_section`, `emit_state_section` (skip roadmap -- not needed for metrics)

3. Emit a `=== TELEMETRY_FILES ===` section listing all `.planning/.telemetry/session-*.jsonl` files found (one per line), or `(none)` if no telemetry directory or files exist.

4. Emit a `=== THRESHOLDS ===` section:
   - If `.planning/.telemetry/config.json` exists, read it and emit each key-value pair as `key=value` lines
   - If missing, emit these defaults:
     ```
     max_tool_calls_per_session=500
     max_agent_spawns_per_session=20
     max_session_duration_minutes=60
     max_failure_rate_percent=5
     ```

5. Emit a `=== SESSION_METRICS ===` section. For each session JSONL file, parse the events and emit one summary line per session in this pipe-delimited format:
   ```
   {session_file}|tools={N}|agents={N}|failures={N}|duration_sec={N}|model={model_name}
   ```
   - `tools`: count of `tool_use` events
   - `agents`: count of `agent_spawn` events
   - `failures`: count of `tool_failure` events
   - `duration_sec`: seconds between first and last event timestamps (use `date` to parse ISO timestamps; if only one event, duration=0)
   - `model`: extracted from the first `skill_start` event's data.model field, or `unknown`

6. Emit a `=== TOOL_BREAKDOWN ===` section. Across ALL session files, count occurrences of each tool_name from `tool_use` events. Emit one line per tool sorted by count descending:
   ```
   {tool_name}={count}
   ```

7. Emit a `=== AGENT_BREAKDOWN ===` section. Across ALL session files, count occurrences of each agent_type from `agent_spawn` events. Emit one line per agent type sorted by count descending:
   ```
   {agent_type}={count}
   ```

8. Emit a `=== TOTALS ===` section with aggregate counts:
   ```
   sessions={N}
   total_tools={N}
   total_agents={N}
   total_failures={N}
   total_duration_sec={N}
   ```

Important implementation notes:
- Use `grep` and `sed`/`awk` for JSONL parsing (jq may not be available). Follow the same pattern as telemetry.sh which has both jq and fallback paths.
- Check `command -v jq` and use jq when available, fallback to grep/sed otherwise.
- Duration calculation: parse ISO timestamps with `date -j -f "%Y-%m-%dT%H:%M:%SZ"` on macOS. If date parsing fails, emit `duration_sec=0`.
- The script must be completely fail-safe -- if any section fails, emit `(error)` and continue to the next section.
- Make the script executable (chmod +x).


**Create SKILL.md** at `plugins/claude-super-team/skills/metrics/SKILL.md`.

Frontmatter:
```yaml
---
name: metrics
description: "Analyze telemetry data to report resource usage per session, tool/agent breakdowns, and threshold violations. Reads .planning/.telemetry/ session files and config.json thresholds."
allowed-tools: Read, Glob, Bash(bash *gather-data.sh)
model: haiku
context: fork
---
```

Key design decisions for the SKILL.md body:
- Model is haiku (this is a reader/reporter -- reads structured data, formats output, no reasoning needed)
- `context: fork` so it does not pollute the parent conversation
- Allowed tools are minimal: Read (for config.json if needed), Glob (not strictly needed but harmless), and Bash for the gather script only
- No Write/Edit tools -- the skill displays the report in the conversation, does not write METRICS.md (per Claude's discretion from CONTEXT.md, keeping it simple)

SKILL.md body structure:

Step 0: Run gather-data.sh via `bash "${CLAUDE_PLUGIN_ROOT}/skills/metrics/gather-data.sh"`. Parse all output sections.

Step 1: Validate environment. If TELEMETRY_FILES section shows `(none)`, display error: "No telemetry data found. Telemetry is captured automatically when using skills from the claude-super-team plugin. Run some skills first, then try /metrics again." Exit.

Step 2: Parse SESSION_METRICS lines into a table. For each session, compute:
- Duration in human-readable format (Xm Ys)
- Failure rate per session: failures / (tools + failures) * 100

Step 3: Threshold violation detection. Compare each session's metrics against THRESHOLDS:
- `max_tool_calls_per_session`: flag if session tools > threshold
- `max_agent_spawns_per_session`: flag if session agents > threshold
- `max_session_duration_minutes`: flag if session duration_sec / 60 > threshold
- `max_failure_rate_percent`: flag if session failure rate > threshold

Collect all violations into a violations list.

Step 4: Generate report following the report-template.md format. Read the template from `${CLAUDE_SKILL_DIR}/assets/report-template.md` for structure reference.

Step 5: Display the report. Do NOT write to a file. Present directly in conversation output.

The SKILL.md must reference the template path as: `${CLAUDE_SKILL_DIR}/assets/report-template.md`
