For each SKILL.md in plugins/claude-super-team/skills/ (16 skills) and the phase-researcher agent, perform this audit:

1. Read the `allowed-tools` line in the YAML frontmatter (or `tools` for agents)
2. Read the entire SKILL.md body (or agent .md body)
3. Identify every tool actually referenced in the instructions:
   - Direct tool mentions: Read, Write, Edit, Glob, Grep, Task, AskUserQuestion, Skill, Bash(...)
   - Tool references in code blocks or instructions: "use Read to...", "spawn via Task tool", "Edit tool to update"
   - Bash subcommand patterns: any `Bash(...)` invocations or bash code blocks showing command patterns
   - For agents: tools declared in `tools:` field vs tools used in agent body
4. Compare declared vs used:
   - **Missing**: Tool used in body but not in allowed-tools -> ADD it
   - **Excess**: Tool in allowed-tools but never referenced in body -> REMOVE it (unless it's a reasonable implicit need)
   - **Correct**: Tool in both -> no change

- **discuss-phase**: Uses `Bash(mkdir *)` for phase dir creation and `Bash(bash *gather-data.sh)` for context loading. Verify Bash patterns cover all inline bash code blocks. Check if `Bash(grep *)` is needed for the ROADMAP.md phase validation grep.
- **plan-phase**: Uses Task for planner spawning, `Bash(cat *)` for reading file contents. Check if `Bash(mkdir *)` is needed now that create_phase_dir() handles directory creation via phase-utils.sh (sourced in a bash call).
- **execute-phase**: Has extensive allowed-tools including TaskCreate, TaskUpdate, TaskGet, TaskList, TaskOutput, TaskStop, TeamCreate, TeamDelete, SendMessage. Verify ALL of these are actually used in the SKILL.md body or referenced guides.
- **build**: Has very broad Bash permissions (npm, npx, bun, pnpm, yarn, cargo, go, make, python, pytest, find, grep, cat, chmod). Verify these are all needed -- build invokes child skills and may need to run project-level build commands.
- **code**: Has `Bash(git diff *)`, `Bash(git status)` -- verify these are actually used in the skill body.
- **cst-help**: Has `Bash(find *)` -- verify this is actually used.
- **new-project**: Has `Bash(git *)`, `Bash(find *)` -- verify usage.
- **map-codebase**: Has `Bash(rm *)` and `Bash(wc *)` -- verify these are needed.
- **brainstorm**: Has `Bash(cat *)` -- verify if this is needed or if Read tool suffices.
- **progress**: Has no Write/Edit -- verify it doesn't need to fix stale state.
- **phase-researcher agent**: Has `Bash` (unrestricted) -- check if this should be more restricted.

For each skill/agent, record: skill name, current permissions, findings (missing/excess/correct), recommended change.
