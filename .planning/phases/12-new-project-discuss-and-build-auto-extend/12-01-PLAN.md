## Overview

Two features that improve the entry-point experience: (1) /new-project gains a `--discuss` flag for open-ended brainstorming sessions that produce PROJECT.md without requiring upfront input, and (2) /build detects existing .planning/ state (PROJECT.md, ROADMAP.md) and auto-routes to extend mode even when no BUILD-STATE.md exists.

## must_haves

- /new-project accepts `--discuss` flag and runs interactive brainstorming to produce PROJECT.md
- /build detects PROJECT.md + ROADMAP.md without BUILD-STATE.md and enters extend mode automatically
- /build handles partial state (PROJECT.md exists, no ROADMAP.md) by skipping /new-project and running from /brainstorm onward
- gather-data.sh emits the signals /build needs to distinguish all five entry states
- All existing paths (fresh, resume, extend with BUILD-STATE.md) remain unchanged

<task id="2" title="Update /build gather-data.sh and SKILL.md branching for auto-extend">

**files:**
- `plugins/claude-super-team/skills/build/gather-data.sh`
- `plugins/claude-super-team/skills/build/SKILL.md`

**action:**

Read both files first. Then make these edits:

**gather-data.sh -- Expand the EXTEND section** to emit finer-grained signals. Currently it only checks BUILD-STATE.md status=complete + PROJECT.md + ROADMAP.md. Add new variables:

```
# After existing EXTEND_CANDIDATE logic:
# Detect partial project state (no BUILD-STATE.md but planning artifacts exist)
if [ "$HAS_PROJECT" = "true" ] && [ "$HAS_ROADMAP" = "true" ] && [ ! -f .planning/BUILD-STATE.md ]; then
  echo "AUTO_EXTEND=true"
elif [ "$HAS_PROJECT" = "true" ] && [ "$HAS_ROADMAP" = "false" ] && [ ! -f .planning/BUILD-STATE.md ]; then
  echo "PARTIAL_PROJECT=true"
else
  echo "AUTO_EXTEND=false"
  echo "PARTIAL_PROJECT=false"
fi
```

**SKILL.md Step 1 -- Add two new branches** between the existing Branch 2 (EXTEND) and Branch 3 (FRESH START). The new branching order becomes:

1. **Branch 1 -- RESUME** (unchanged): BUILD-STATE.md exists with status=in_progress
2. **Branch 2 -- EXTEND** (unchanged): EXTEND_CANDIDATE=true AND $ARGUMENTS non-empty
3. **Branch 2a -- AUTO-EXTEND** (NEW): `AUTO_EXTEND=true` from gather-data.sh AND $ARGUMENTS is non-empty. This is the case where PROJECT.md + ROADMAP.md exist but no BUILD-STATE.md. Behavior: Print "Auto-extend mode: existing project detected, adding feature." then skip to Step 1-E (same extend entry point).
4. **Branch 2b -- PARTIAL PROJECT** (NEW): `PARTIAL_PROJECT=true` from gather-data.sh AND $ARGUMENTS is non-empty. This is PROJECT.md exists but no ROADMAP.md. Behavior: Print "Partial project detected: skipping /new-project, starting from brainstorm." Then skip to Step 3 (Load Preferences), but in Step 4 (Initialize BUILD-STATE.md) mark the `new-project` pipeline row as `skipped` with Notes "Pre-existing PROJECT.md", and in Step 5 check: if `HAS_PROJECT=true` from gather-data.sh, skip the /new-project invocation and set pipeline row to `skipped`. Continue normally from Step 7 (/brainstorm) onward.
5. **Branch 3 -- FRESH START** (unchanged): Everything else

Also update the error message in Step 2 for empty $ARGUMENTS. Currently it says "/build requires a project idea or path to a PRD file." Add: "If you have an existing project, provide the feature you want to add." This helps users who invoke /build on an existing project without arguments understand what to provide.
