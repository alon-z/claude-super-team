<objective>
Reduce plan-phase time-to-first-agent-spawn by moving context assembly from LLM processing into gather-data.sh, so the skill LLM receives pre-trimmed, ready-to-embed context sections and can proceed to Task spawn with minimal file-reading overhead.

Purpose: Addresses the highest-impact gap (plan-phase startup speed) identified during workflow validation. Simple phases currently take the same startup time as complex ones because the LLM reads and reasons through all context files sequentially before spawning the planner agent.
Output: Updated gather-data.sh with pre-assembled context sections, streamlined SKILL.md, updated context-loading.md reference.
</objective>

  <name>Task 1: Expand gather-data.sh to pre-assemble planner context</name>
  <files>plugins/claude-super-team/skills/plan-phase/gather-data.sh</files>
  <action>
Expand gather-data.sh to pre-assemble trimmed context sections that the skill LLM currently builds by reading multiple files and reasoning about what to include. The script already outputs PROJECT, ROADMAP, STATE, PHASE_STATUS, and ROADMAP_PHASES sections. Add new sections that do the trimming work in bash:

**New section: ROADMAP_TRIMMED**
Extract only the `## Phases` list (one-liners) and the specific `### Phase N` detail block for the target phase. This replaces the LLM reading the full ROADMAP.md and deciding which parts to send to the planner.

Accept a `PHASE_NUM` environment variable (the raw phase number, e.g., "5" or "2.1"). If not set, output "(no phase specified -- use full roadmap)". If set:
- Extract lines from `## Phases` through the next `##` heading (the phase checklist)
- Extract the `### Phase {PHASE_NUM}:` block through the next `###` heading or end of file
- Output both under `=== ROADMAP_TRIMMED ===`

**New section: STATE_TRIMMED**
Extract only `## Current Position` through the next `##` heading, and `## Preferences` through the next `##` heading, and `## Accumulated Context` (which contains Key Decisions) through the next `##` heading. Skip execution history/logs. This replaces the LLM reading full STATE.md and deciding what to trim.

If `SKIP_STATE=1`, output "(in context)".

**New section: CODEBASE_DOCS**
If `.planning/codebase/` directory exists, concatenate the content of ARCHITECTURE.md, STACK.md, CONVENTIONS.md, and STRUCTURE.md (the four most planning-relevant docs) with clear section headers. If `SKIP_CODEBASE=1`, output "(in context)". If the directory doesn't exist, output "(no codebase docs)".

**New section: PHASE_CONTEXT**
Accept `PHASE_DIR` environment variable. If set and a `*-CONTEXT.md` file exists in that directory, output its content. Otherwise output "(none)".

**New section: PHASE_RESEARCH**
Same pattern as PHASE_CONTEXT but for `*-RESEARCH.md`.

**New section: PHASE_REQUIREMENTS**
If `.planning/REQUIREMENTS.md` exists, output its content. Otherwise output "(none)".

Keep all existing sections (PROJECT, ROADMAP, STATE, PHASE_STATUS, ROADMAP_PHASES) unchanged for backward compatibility. The new sections are additive. The existing ROADMAP and STATE sections are still needed for Phase 1 validation (checking they exist) and for --all mode discovery.

Add SKIP_CODEBASE to the skip flags pattern alongside the existing SKIP_PROJECT, SKIP_ROADMAP, SKIP_STATE.

Important: The bash trimming doesn't need to be perfect -- approximate extraction with sed/awk is fine. The planner agent can handle slightly imprecise boundaries. The goal is to avoid the LLM doing 6+ sequential Read tool calls before reaching the Task spawn.
  </action>
