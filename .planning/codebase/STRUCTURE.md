## Directory Purposes

**`.claude-plugin/`:**
- Purpose: Marketplace-level configuration
- Contains: `marketplace.json` (plugin registry manifest)
- Key files: `.claude-plugin/marketplace.json` (source of truth for installed plugins)

**`plugins/`:**
- Purpose: Modular plugin distribution units
- Contains: Plugin subdirectories with isolated skills, agents, metadata
- Key files: Each plugin has `.claude-plugin/plugin.json`, `skills/` directory, optional `agents/` directory

**`plugins/claude-super-team/`:**
- Purpose: Core project planning and execution workflow
- Contains: 13 skills, 1 custom agent, references, assets, templates
- Key files:
  - `.claude-plugin/plugin.json` (plugin metadata)
  - `agents/phase-researcher.md` (custom researcher agent)
  - `skills/{skill-name}/SKILL.md` (13 skill definitions)
  - `skills/{skill-name}/assets/*.md` (artifact templates)
  - `skills/{skill-name}/references/*.md` (agent instructions)

**`plugins/claude-super-team/skills/`:**
- Purpose: Individual slash command implementations
- Contains: 13 skill directories (new-project, map-codebase, create-roadmap, brainstorm, discuss-phase, research-phase, plan-phase, execute-phase, progress, quick-plan, phase-feedback, add-security-findings, cst-help)
- Key files: Each skill has `SKILL.md`, optional `assets/` and `references/` subdirectories

**`plugins/claude-super-team/agents/`:**
- Purpose: Reusable subagent definitions
- Contains: Custom agent markdown files
- Key files: `phase-researcher.md` (ecosystem/library research agent)

**`plugins/marketplace-utils/`:**
- Purpose: Marketplace management and skill scaffolding
- Contains: 2 skills (marketplace-manager, skill-creator)
- Key files:
  - `skills/marketplace-manager/SKILL.md` (audit, register, sync plugins)
  - `skills/skill-creator/SKILL.md` (scaffold new skills)
  - `skills/skill-creator/scripts/validate_yaml.py` (YAML validation script)

**`plugins/task-management/`:**
- Purpose: External task tracker integrations
- Contains: 2 skills (linear-sync, github-issue-manager)
- Key files:
  - `skills/linear-sync/SKILL.md` (sync to Linear)
  - `skills/github-issue-manager/SKILL.md` (GitHub issue management)

**`.planning/`:**
- Purpose: User project planning artifacts (created during skill execution)
- Contains: PROJECT.md, ROADMAP.md, STATE.md, phases/*, codebase/* (generated by skills in user projects)
- Generated: Yes (created when skills run in user projects)
- Committed: Yes (planning artifacts are version-controlled)

**`.planning/codebase/`:**
- Purpose: Codebase analysis documents (created by `/map-codebase`)
- Contains: STACK.md, INTEGRATIONS.md, ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Generated: Yes (written by mapper agents)
- Committed: Yes (codebase understanding is versioned)

**`.planning/phases/`:**
- Purpose: Per-phase planning and execution artifacts
- Contains: Directories named `{NN}-{phase-name}/` with CONTEXT, RESEARCH, PLAN, SUMMARY, VERIFICATION files
- Generated: Yes (one directory per roadmap phase)
- Committed: Yes (phase artifacts track project history)

## Key File Locations

**Entry Points:**
- `.claude-plugin/marketplace.json`: Marketplace root (plugin discovery)
- `plugins/{name}/.claude-plugin/plugin.json`: Plugin manifest (skill registration)
- `plugins/{name}/skills/{skill}/SKILL.md`: Skill definition (slash command)

**Configuration:**
- `.claude-plugin/marketplace.json`: Marketplace plugin registry
- `plugins/{name}/.claude-plugin/plugin.json`: Plugin metadata (name, version, description, keywords)
- Skill frontmatter in `SKILL.md`: Per-skill configuration (allowed-tools, model, context)

**Core Logic:**
- `plugins/claude-super-team/skills/new-project/SKILL.md`: Project initialization
- `plugins/claude-super-team/skills/map-codebase/SKILL.md`: Codebase mapping orchestrator
- `plugins/claude-super-team/skills/execute-phase/SKILL.md`: Wave execution orchestrator
- `plugins/claude-super-team/agents/phase-researcher.md`: Research agent

**Testing:**
- Not detected (no test/ or tests/ directories)
- No test framework configuration files (no jest.config.js, pytest.ini, etc.)

**Documentation:**
- `README.md`: User-facing installation and workflow guide
- `CLAUDE.md`: Project instructions for Claude (conventions, structure, workflow)
- `ORCHESTRATION-REFERENCE.md`: Multi-agent patterns reference
- `CHANGELOG.md`: Version history
- `plugins/{name}/README.md`: Plugin-specific documentation (exists for claude-super-team, task-management)

## Naming Conventions

**Files:**
- Plugin manifests: `plugin.json` (all plugins)
- Skill definitions: `SKILL.md` (all skills use this exact name)
- Templates: Lowercase kebab-case (e.g., `roadmap.md`, `plan-template.md`)
- Guides: Lowercase kebab-case (e.g., `planner-guide.md`, `task-execution-guide.md`)
- Agents: Lowercase kebab-case (e.g., `phase-researcher.md`)
- Generated artifacts: UPPERCASE.md (e.g., `PROJECT.md`, `ROADMAP.md`, `STATE.md`, `RESEARCH.md`, `PLAN.md`, `SUMMARY.md`)
- Codebase docs: UPPERCASE.md (e.g., `STACK.md`, `ARCHITECTURE.md`, `TESTING.md`)

**Directories:**
- Plugins: Lowercase kebab-case (e.g., `claude-super-team`, `marketplace-utils`)
- Skills: Lowercase kebab-case (e.g., `new-project`, `map-codebase`, `execute-phase`)
- Agents: Lowercase singular (e.g., `agents/`)
- Assets: Lowercase singular (e.g., `assets/`)
- References: Lowercase singular (e.g., `references/`)
- Phases: Zero-padded number + kebab-case name (e.g., `01-foundation`, `02-auth`, `04.1-security-hardening`)

**Phase Numbering:**
- Standard phases: `01`, `02`, `03`, etc. (zero-padded)
- Inserted phases: Decimal notation (e.g., `02.1`, `04.1`) for gap closure or quick plans
- Phase directories: `{NN}-{name}` format (e.g., `01-foundation`)
- Phase files: `{NN}-{suffix}.md` format (e.g., `01-CONTEXT.md`, `01-01-PLAN.md`)

## Where to Add New Code

**New Skill (claude-super-team):**
- Primary code: `plugins/claude-super-team/skills/{skill-name}/SKILL.md`
- Templates: `plugins/claude-super-team/skills/{skill-name}/assets/{template}.md`
- Agent instructions: `plugins/claude-super-team/skills/{skill-name}/references/{guide}.md`
- Update: `/cst-help` skill reference section, `CLAUDE.md` workflow section, `CHANGELOG.md`

**New Custom Agent:**
- Implementation: `plugins/claude-super-team/agents/{agent-name}.md`
- Reference from: Skills using Task tool with agent parameter

**New Plugin:**
- Implementation: `plugins/{plugin-name}/` directory
- Plugin manifest: `plugins/{plugin-name}/.claude-plugin/plugin.json`
- Skills: `plugins/{plugin-name}/skills/{skill-name}/SKILL.md`
- Register: Add entry to `.claude-plugin/marketplace.json` via `/marketplace-manager`
- Update: `README.md` plugins table, `CHANGELOG.md`

**New Template:**
- Location: `plugins/{plugin-name}/skills/{skill-name}/assets/{template-name}.md`
- Reference: Embed in SKILL.md body or agent definition

**New Reference Guide:**
- Location: `plugins/{plugin-name}/skills/{skill-name}/references/{guide-name}.md`
- Reference: Embed in SKILL.md body when spawning agents via Task tool

**Utilities/Helpers:**
- Shared helpers: Not applicable (no shared code layer; each skill is self-contained)
- Scripts: `plugins/{plugin-name}/skills/{skill-name}/scripts/` (e.g., `validate_yaml.py` in skill-creator)

## Special Directories

**`.claude-plugin/`:**
- Purpose: Claude Code plugin system metadata
- Generated: No (manually created)
- Committed: Yes (required for plugin discovery)

**`.planning/`:**
- Purpose: User project planning state
- Generated: Yes (created by skills during execution)
- Committed: Yes (tracks project planning history)

**`assets/` (per skill):**
- Purpose: Markdown templates for artifact generation
- Generated: No (authored by skill creators)
- Committed: Yes (part of skill distribution)

**`references/` (per skill):**
- Purpose: Agent instruction guides embedded into prompts
- Generated: No (authored by skill creators)
- Committed: Yes (part of skill distribution)

**`agents/`:**
- Purpose: Custom subagent definitions
- Generated: No (authored manually)
- Committed: Yes (reusable agent configurations)

**`scripts/` (per skill):**
- Purpose: Helper scripts invoked by skills
- Generated: No (authored by skill creators)
- Committed: Yes (executable dependencies for skills)
- Example: `plugins/marketplace-utils/skills/skill-creator/scripts/validate_yaml.py`

**`.claude/`:**
- Purpose: User-specific Claude Code settings (not part of repository distribution)
- Generated: No (user environment)
- Committed: No (local configuration only)
