**Connector proxy API route (src/app/api/connectors/route.ts):**
Create a Next.js API route that proxies connector management to the ingestion service:
- POST /api/connectors: Accept { type: "stripe", config: { apiKey: string }, refreshInterval: string }. Read workspace ID from the authenticated session. Forward to INGESTION_SERVICE_URL/api/connectors with the workspace_id added. Pass INGESTION_API_KEY as authorization. Return the created data source record.
- GET /api/connectors: List data sources for the current workspace. Read workspace ID from session. Proxy to ingestion service.

**Stripe connection page (connect-stripe/page.tsx):**
Client component. A simple form with:
- Text input for Stripe API key (type password to mask it)
- Select dropdown for refresh interval: 1 minute, 5 minutes, 15 minutes, manual
- "Connect Stripe" submit button
- On submit, POST to /api/connectors with type "stripe" and the form values
- On success, show a success message and redirect to /dashboard after 2 seconds
- On error (e.g., invalid API key -- the ingestion service validates), show the error message
- Brief instructions: "Enter your Stripe secret key (starts with sk_). Find it in Stripe Dashboard > Developers > API keys."

Link to this page from the main dashboard when no data sources are connected yet.
  </action>

Navigate back to /dashboard/connect-stripe and verify the form still works for adding another source. Test with an invalid key and confirm an error is displayed.
  </verify>

**Dashboard config (dashboard-config.ts):**
Define a DashboardConfig type: { widgets: WidgetConfig[] } where WidgetConfig is { id: string, type: "line-chart" | "bar-chart" | "kpi-card", title: string, metricName: string, position: { row: number, col: number, colSpan: number }, props: Record<string, unknown> }.

Create a defaultDashboard configuration that defines the initial widget layout for a Stripe-connected workspace:
- Row 1: Three KPI cards spanning the full width (4 cols each in a 12-col grid)
  - "Current Balance" (metric: balance, format: currency)
  - "Total Revenue (30d)" (metric: daily_revenue, format: currency, trendPeriod: week)
  - "Charges (30d)" (metric: charge_count, format: number, trendPeriod: week)
- Row 2: Line chart spanning 8 cols -- "Revenue Over Time" (metric: daily_revenue, last 30 days)
- Row 2: Bar chart spanning 4 cols -- "Daily Charges" (metric: charge_count, bucketBy: day, last 30 days)

**Dashboard page (src/app/(app)/dashboard/page.tsx):**
Replace the placeholder from Phase 1 Plan 02. This is a mixed server/client page:

Server portion: Fetch the user's data sources via the /api/connectors endpoint (server-side fetch). If no data sources exist, show an empty state with a prominent "Connect Stripe" button linking to /dashboard/connect-stripe. If data sources exist, render the dashboard.

Client portion (separate client component): Accept the data source list and dashboard config. Render a CSS Grid layout (12 columns) that places widgets according to their position config. For each widget config, render the appropriate component (LineChart, BarChart, or KpiCard) with the sourceId from the first connected Stripe source and the configured metricName.

Auto-refresh: Each widget handles its own refresh internally (from Plan 01 -- they accept refreshInterval prop). Pass the data source's configured refresh interval to each widget. Additionally, add a manual "Refresh All" button in the dashboard header that forces all widgets to re-fetch.

Dashboard header should show: workspace name, "Refresh All" button, and a "Connect Data Source" button.
