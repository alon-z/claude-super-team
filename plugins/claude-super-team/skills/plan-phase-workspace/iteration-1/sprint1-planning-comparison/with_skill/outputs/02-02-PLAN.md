Install the stripe npm package.

**Connector interface (types.ts):**
Define a Connector interface that all data source connectors must implement:
- name: string (e.g., "stripe")
- sync(sourceConfig: Record<string, unknown>, since: Date): Promise<MetricRecord[]> -- fetches metrics since the given timestamp
- validate(sourceConfig: Record<string, unknown>): Promise<boolean> -- validates the provided credentials

Define MetricRecord type: { time: Date, metricName: string, metricValue: number, dimensions: Record<string, string> }

**Stripe connector (stripe.ts):**
Implement the Connector interface for Stripe. The sourceConfig will contain { apiKey: string }.

The sync() method should:
1. Initialize a Stripe client with the provided API key
2. Fetch charges (list charges created after `since` date, paginate if needed with auto-pagination)
3. Aggregate into metrics: daily revenue (sum of charge amounts), charge count, average charge amount
4. Fetch the current balance and emit as a point-in-time metric
5. Return all metrics as MetricRecord[] with appropriate dimensions (e.g., currency)

The validate() method should:
1. Attempt to call stripe.balance.retrieve() with the provided key
2. Return true if successful, false on auth error

Handle rate limiting: if Stripe returns 429, implement a simple exponential backoff with 3 retries. The Stripe SDK handles this automatically via its retry config -- set maxNetworkRetries to 3 when initializing the client.

Convert Stripe amounts from cents to dollars in the metric values (divide by 100).
  </action>

Write a simple test script that creates a StripeConnector instance with a test API key (use Stripe test mode key from env STRIPE_TEST_KEY). Call validate() and confirm it returns true. Call sync() with a since date of 30 days ago and confirm it returns MetricRecord[] (may be empty in test mode -- that's fine, verify the shape). Confirm that amounts are in dollars, not cents.
  </verify>
  <done>Stripe connector implements the Connector interface. sync() fetches charges and balance from Stripe API and returns MetricRecord[] with daily revenue, charge count, and balance metrics. validate() confirms API key validity. Rate limiting is handled via Stripe SDK retry config.</done>

**Connector routes (connectors.ts):**
Create Hono route group for /api/connectors:
- POST /api/connectors: Register a new data source. Accept { workspaceId, type, config, refreshInterval }. Validate the connector credentials by calling connector.validate(config). If valid, insert into data_sources table and return the created record. If invalid, return 400 with error.
- GET /api/connectors?workspaceId=xxx: List all data sources for a workspace.
- DELETE /api/connectors/:id: Remove a data source and its scheduled jobs.

For now, authentication between the Next.js app and ingestion service uses a shared API key (INGESTION_API_KEY env var) passed as a Bearer token. The routes should check this header.

**Sync routes (sync.ts):**
- POST /api/sync/:sourceId: Manually trigger a sync for a data source. Create a sync_job record with status "pending", run the connector's sync() method, insert the returned MetricRecord[] into the metrics hypertable, update sync_job to "completed" with records_synced count. On failure, update to "failed" with error_message.
- GET /api/sync/:sourceId/status: Return the latest sync_job for a data source.

**Scheduler (scheduler.ts):**
Create a simple interval-based scheduler that:
1. On startup, loads all active data_sources from the database
2. Groups them by refresh_interval
3. Sets up setInterval timers: 60s for "1min", 300s for "5min", 900s for "15min"
4. On each tick, triggers sync for all sources in that interval group by calling the sync route logic internally (not via HTTP -- call the sync function directly)
5. "manual" sources are never auto-synced
6. Exports startScheduler() function that initializes the timer loops

**Update index.ts:**
Mount the connector and sync routes on the Hono app. Call startScheduler() after the server starts listening. Log scheduler activity to console.
