  <action>
Install dependencies: jose (for JWT -- NOT jsonwebtoken, which has CommonJS issues with Next.js Edge runtime), bcryptjs (for password hashing), and arctic (for OAuth providers).

**Auth library (src/lib/auth.ts):**
Create utility functions:
- hashPassword(password): bcryptjs hash with salt rounds 12
- verifyPassword(password, hash): bcryptjs compare
- createToken(payload): Create a JWT using jose with HS256 algorithm, 7-day expiry. Payload should include userId, email, workspaceId. Read JWT_SECRET from env.
- verifyToken(token): Verify and decode JWT using jose. Return payload or null.
- getCurrentUser(request): Extract JWT from httpOnly cookie named "session", verify it, return user payload or null.

**Signup endpoint (POST /api/auth/signup):**
Accept { email, password, name, workspaceName }. Validate all fields present. Check email uniqueness against users table. Hash password. In a transaction: create workspace (generate slug from workspaceName -- lowercase, replace spaces with hyphens, append random 4 chars for uniqueness), create user, create workspace_member with role "owner". Return JWT in httpOnly secure cookie and redirect URL.

**Login endpoint (POST /api/auth/login):**
Accept { email, password }. Look up user by email. Verify password with bcryptjs. If invalid, return 401 with generic "Invalid credentials" message (don't reveal whether email exists). Look up user's workspace membership to get workspaceId. Create JWT and set as httpOnly secure cookie. Return success with redirect URL.

**Google OAuth initiation (GET /api/auth/google):**
Configure Google OAuth using arctic library. Redirect to Google consent screen with scopes: openid, email, profile. Store a random state parameter in a short-lived httpOnly cookie for CSRF protection. Environment variables needed: GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, GOOGLE_REDIRECT_URI.

**Google OAuth callback (GET /api/auth/google/callback):**
Validate state parameter against stored cookie. Exchange authorization code for tokens using arctic. Extract email, name, and Google ID from the ID token. If a user with this email exists, log them in (create JWT, set cookie). If no user exists, create a new user (password_hash = null), create a workspace for them, create workspace_member with role "owner", then create JWT and set cookie. Redirect to /dashboard.

**Update middleware.ts (from Plan 01):**
Enhance the existing middleware to also check authentication. For non-public paths, call getCurrentUser(). If no valid session, redirect to /login. If authenticated, ensure the workspace ID from the JWT matches the resolved workspace context.
  </action>

**Route group layouts:**
Create (public) layout: minimal centered layout with no nav, suitable for auth pages.
Create (app) layout: basic app shell with a sidebar placeholder showing workspace name and a main content area. The workspace name should be read from the JWT payload via getCurrentUser() in a server component.

**Signup page (/signup):**
Client component with a form: name, email, password, workspace name fields. On submit, POST to /api/auth/signup. On success, redirect to /dashboard using next/navigation router.push. On error, display the error message below the form. Include a link "Already have an account? Sign in" pointing to /login. Include a "Sign up with Google" button that navigates to /api/auth/google.

**Login page (/login):**
Client component with a form: email, password fields. On submit, POST to /api/auth/login. On success, redirect to /dashboard. On error, display error message. Include a link "Don't have an account? Sign up" pointing to /signup. Include a "Sign in with Google" button that navigates to /api/auth/google.

**Dashboard page (/dashboard):**
Server component. Use getCurrentUser() to get the authenticated user. Query the workspace name from the database using the workspaceId from the JWT. Display a welcome message: "Welcome to {workspaceName}" with the user's name and email. This is a placeholder that Phase 3 will replace with actual dashboard widgets.

Use basic Tailwind CSS for styling. No component library needed at this stage.
